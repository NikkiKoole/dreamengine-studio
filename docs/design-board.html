THINK-ABOUT (not a bug — current behaviour is correct): TIE and a same-pitch SLIDE are the SAME sound in the common case — verified byte-identical WAV renders (same pitch, no accent: re-asserting the same note_vol + a glide to the same pitch is a true no-op, even across a 1s hold). They only diverge (a) at DIFFERENT pitches — then slide is a real portamento, not a hold — or (b) with ACCENT on the held step, where slide re-runs note_vol → a ~2.8dB mid-note swell a tie can't make (and the UI force-clears a tie step's own note, so a tie can't carry accent anyway). So is the TIE row redundant? No — it earns its place on ERGONOMICS, especially LIVE: with a tie you lengthen a note as a single event, so while the loop plays you can grab that one note and re-pitch it on the fly — the hold follows. The slide-fake is two pitch-matched steps, so re-pitching live means editing both in lockstep or it decays into a glide; you can't tweak a long note quickly mid-performance. (Also pitch-agnostic + edit-stable in the static case.) Leaning keep; parked here in case the flag rows ever get simplified.
