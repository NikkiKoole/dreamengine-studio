# ios/ — Tinyjam native iOS scaffold

The reusable, **Xcode-GUI-free** build loop for shipping carts as native iOS apps.
Rationale, decisions, and the spike roadmap: [`../docs/design/ios-plan.md`](../docs/design/ios-plan.md).
Why we ship apps (not the editor): [ADR-0023](../docs/decisions/0023-ship-carts-as-apps-not-the-editor.md).

## Prerequisites (one-time)

- **Full Xcode active** — `xcode-select -p` must point at `Xcode*.app/Contents/Developer`, NOT
  `/Library/Developer/CommandLineTools`. Set it (the bit that needs your password):
  ```bash
  sudo xcode-select -s /path/to/Xcode.app/Contents/Developer
  sudo xcodebuild -license accept
  xcodebuild -runFirstLaunch
  ```
- **xcodegen** — `brew install xcodegen`.

Everything below is **simulator-only: no code-signing, no Apple-account interaction, free.**
(Only AUv3 — spike 7 — will need a physical device.)

## Run the demo yourself (terminal)

```bash
cd ios
./build.sh                     # generate project → build → boot simulator → install → launch
```

That builds the app and launches it on the **iPhone 15** simulator. You'll see the lo-fi canvas
(pink ball), hear the arpeggio, and see the green VU + store-gate dot. Variations:

```bash
SHOT=snap.png ./build.sh                        # also save a screenshot to snap.png
DEVICE="iPhone SE (3rd generation)" ./build.sh   # use a different simulator
xcrun simctl list devices available              # see all installed simulators
```

**See the simulator window** (it opens in the background):
```bash
open -a Simulator                                # bring it to the front (also ⌘-Tab → "Simulator")
```

**Stop the sound / app** (the arpeggio loops forever by design):
```bash
xcrun simctl terminate "iPhone 15" com.tinyjam.hello   # quit the app, keep the sim up
xcrun simctl shutdown "iPhone 15"                       # shut the whole simulator down
```

**Take a screenshot anytime** (the app keeps running):
```bash
xcrun simctl io "iPhone 15" screenshot shot.png
```

## Run the StoreKit tests yourself

Proves in-app purchases with **no Apple account and no network** (a local `Tinyjam.storekit`
config via the StoreKitTest framework):
```bash
cd ios
xcodebuild test -project TinyjamHello.xcodeproj -scheme TinyjamHello \
  -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 15' \
  CODE_SIGNING_ALLOWED=NO
```
Expect `Test Suite 'StoreTests' passed` (buys a rack → entitlement unlocks; master pass unlocks all).

## Files

- `project.yml` — the xcodegen spec; the **only** hand-edited project file. Add targets/sources here.
- `Sources/` — app source: `App.swift` (SwiftUI shell), `CanvasView.swift` (framebuffer → screen),
  `canvas.{h,c}` (the software canvas), `audio.{h,c}` + `AudioEngine.swift` (CoreAudio synth),
  `Store.swift` + `tinyjam_store.h` (StoreKit 2 in-house bridge), `Tinyjam.storekit` (local IAP
  config), `save.{h,c}` + `Save.swift` (Documents-dir save layer), `Bridging-Header.h` (exposes the
  C APIs to Swift).
- `Tests/` — `StoreTests.swift` (headless IAP proof via SKTestSession), `SaveTests.swift` (save
  round-trip).
- `build.sh` — the one-command run loop above.
- `history/` — committed milestone screenshots (the visual changelog; see `history/README.md`).
- `.gitignore` — the generated `*.xcodeproj/` and `build/` are never committed (regenerated by `build.sh`).
