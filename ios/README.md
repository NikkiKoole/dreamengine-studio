# ios/ — Tinyjam native iOS scaffold

The reusable, **Xcode-GUI-free** build loop for shipping carts as native iOS apps.
Rationale, decisions, and the spike roadmap: [`../docs/design/ios-plan.md`](../docs/design/ios-plan.md).
Why we ship apps (not the editor): [ADR-0023](../docs/decisions/0023-ship-carts-as-apps-not-the-editor.md).

## Build & run (one command)

```bash
cd ios
./build.sh                                   # generate → build → boot sim → install → launch
SHOT=snap.png ./build.sh                      # …and screenshot to snap.png
DEVICE="iPhone SE (3rd generation)" ./build.sh
```

Simulator-only: **no code-signing, no Apple-account interaction.** Everything through spike 6 runs
this way for free; only AUv3 (spike 7) needs a physical device.

## Prerequisites (one-time)

- **Full Xcode active** — `xcode-select -p` must point at `Xcode*.app/Contents/Developer`, NOT
  `/Library/Developer/CommandLineTools`. Set with `sudo xcode-select -s /path/to/Xcode.app/Contents/Developer`
  (then `sudo xcodebuild -license accept` + `xcodebuild -runFirstLaunch`).
- **xcodegen** — `brew install xcodegen`.

## Files

- `project.yml` — the xcodegen spec; the **only** hand-edited project file. Add targets/sources here.
- `Sources/` — app source. Currently the spike-0 SwiftUI hello-world; spike 1 replaces it with a
  view hosting the software-canvas framebuffer.
- `build.sh` — the loop above.
- `.gitignore` — the generated `*.xcodeproj/` and `build/` are never committed (regenerated by `build.sh`).
