# ios/ — Tinyjam native iOS scaffold

The reusable, **Xcode-GUI-free** build loop for shipping carts as native iOS apps.
Rationale, decisions, and the spike roadmap: [`../docs/design/ios-plan.md`](../docs/design/ios-plan.md).
Why we ship apps (not the editor): [ADR-0023](../docs/decisions/0023-ship-carts-as-apps-not-the-editor.md).

## Prerequisites (one-time)

- **Full Xcode active** — `xcode-select -p` must point at `Xcode*.app/Contents/Developer`, NOT
  `/Library/Developer/CommandLineTools`. Set it (the bit that needs your password):
  ```bash
  sudo xcode-select -s /path/to/Xcode.app/Contents/Developer
  sudo xcodebuild -license accept
  xcodebuild -runFirstLaunch
  ```
- **xcodegen** — `brew install xcodegen`.

Everything below is **simulator-only: no code-signing, no Apple-account interaction, free.**
(Only AUv3 — spike 7 — will need a physical device.)

## Run the demo yourself (terminal)

```bash
cd ios
./build.sh                     # generate project → build → boot simulator → install → launch
```

That builds the app and launches it on the **iPhone 15** simulator. You'll see the lo-fi canvas
(pink ball), hear the arpeggio, and see the green VU + store-gate dot. Variations:

```bash
SHOT=snap.png ./build.sh                        # also save a screenshot to snap.png
DEVICE="iPhone SE (3rd generation)" ./build.sh   # use a different simulator
xcrun simctl list devices available              # see all installed simulators
```

**See the simulator window** (it opens in the background):
```bash
open -a Simulator                                # bring it to the front (also ⌘-Tab → "Simulator")
```

**Stop the sound / app** (the arpeggio loops forever by design):
```bash
xcrun simctl terminate "iPhone 15" com.tinyjam.hello   # quit the app, keep the sim up
xcrun simctl shutdown "iPhone 15"                       # shut the whole simulator down
```

**Take a screenshot anytime** (the app keeps running):
```bash
xcrun simctl io "iPhone 15" screenshot shot.png
```

## Run on a real iPhone

```bash
cd ios
./device.sh                      # build signed → install → launch on the connected iPhone
```

One-time setup: add your Apple ID in **Xcode → Settings → Accounts** (mints the signing cert),
`brew install ios-deploy`, and connect + unlock the phone. We use `ios-deploy` (not `devicectl`)
because CoreDevice doesn't enroll older devices (iOS 15) without a GUI step. First launch may need:
iPhone → Settings → General → VPN & Device Management → trust the "Apple Development" profile.

## Run the StoreKit tests yourself

Proves in-app purchases with **no Apple account and no network** (a local `Tinyjam.storekit`
config via the StoreKitTest framework):
```bash
cd ios
xcodebuild test -project TinyjamHello.xcodeproj -scheme TinyjamHello \
  -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 15' \
  CODE_SIGNING_ALLOWED=NO
```
Expect `Test Suite 'StoreTests' passed` (buys a rack → entitlement unlocks; master pass unlocks all).

## Files

- `project.yml` — the xcodegen spec; the **only** hand-edited project file. Add targets/sources here.
- `Sources/` — app source: `App.swift` (SwiftUI shell), `CanvasView.swift` (engine framebuffer →
  screen + touch), `AudioEngine.swift` (CoreAudio → the real `de_audio_render`), `engine.h` (the
  platform-seam decls the bridging header exposes to Swift), `Store.swift` + `tinyjam_store.h`
  (StoreKit 2 in-house bridge), `Tinyjam.storekit` (local IAP config), `save.{h,c}` + `Save.swift`
  (Documents-dir save layer), `Bridging-Header.h`. **`canvas.{h,c}` / `audio.{h,c}` are the spike
  stand-ins — excluded from the app now (the real engine replaced them); kept only for the AUv3 target.**
- `AU/` — the AUv3 instrument extension: `TinyjamAU.swift` (`AUAudioUnit` whose render block calls
  the shared C synth) + bridging header. Registration is generated into `AU/Info.plist` by xcodegen.
- `Tests/` — `StoreTests.swift` (headless IAP via SKTestSession), `SaveTests.swift` (save round-trip),
  `AppGroupTests.swift` (entitlement sharing), `AUHostTests.swift` (our own host: finds + renders the
  AUv3, no GarageBand/device needed).
- `build.sh` — the one-command run loop above.
- `history/` — committed milestone screenshots (the visual changelog; see `history/README.md`).
- `.gitignore` — the generated `*.xcodeproj/` and `build/` are never committed (regenerated by `build.sh`).

## Phase 2 — the real engine (omnichord), not the spike stand-ins

The app now hosts the **real dreamengine** (`runtime/studio.c` + `sound.h` + a cart), not the spike
stand-in `canvas.c`/`audio.c`. `./build.sh` first regenerates the cart (`build/cart.c` + data headers)
via `play.js`, then builds it into the app with `-DDE_NO_RAYLIB`. Swap the cart without touching code:

```bash
cd ios
CART=omnichord ./build.sh          # the default; any tools/carts/<name>.c works
SHOT=snap.png CART=tb303 ./build.sh # build a different cart's app + screenshot
```

`CanvasView` blits the engine's software framebuffer (`de_framebuffer()`, flipped — `sw_cbuf` is
bottom-up) and feeds UIKit touches to `de_touch_*`; `AudioEngine` pulls the real mixer
(`de_audio_render`). The desktop twin of this build is `../tools/build-nr.sh`.

**The AUv3 extension is a playable instrument rack** hosting the real engine (a separate cart —
`AU_CART`, default `epiano`, a keybed instrument). `AU/TinyjamAU.swift`'s render block parses the
host's MIDI event list → `de_midi_event()`, sample-clocks `de_frame()` (the cart's keybed plays the
notes), then pulls `de_audio_render()` — correct even under a host's offline render, and all on the
one audio thread (no MIDI-ring locking). Verified by `AUHostTests`: silent with no MIDI (peak 0.000),
then a host note-on → peak 0.106. Engine seam: `midi_input.h` exposes `de_midi_event`/`de_midi_bend`
under `DE_NO_RAYLIB` (portable host-feed, like the web bridge). Next: MIDI CC → cart knobs.
